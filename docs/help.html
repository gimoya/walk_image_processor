🚀 Schnellstart

Perfekter Workflow für Ihre Wegdokumentation:
  1. Navigieren Sie zu Ihrem Bilder-Ordner
  2. Öffnen Sie die Eingabeaufforderung in diesem Ordner
  3. Führen Sie wip aus
  4. Erhalten Sie eine professionelle Wegdokumentation

Basis-Befehl

# Einfachste Verwendung wip # Mit benutzerdefinierten Einstellungen wip -t "Meine Wanderung" -d "15-03-2025" -l "Alpenregion"

📋 Detaillierter Workflow

1. Vorbereitung der Bilder

Ihre Bilder können verschiedene Formate haben:
001_startpunkt___13.202000_47.316875___.jpg (mit Zeitstempel + Koordinaten)
landschaft___13.205002_47.315930___.jpg (mit Koordinaten)
gipfel___13.207121_47.316326___.png (mit Koordinaten)
overview.jpg (ohne Zeitstempel/Koordinaten - wird trotzdem verarbeitet)

2. Navigation zum Bildordner

# Windows Explorer: Shift + Rechtsklick → "Eingabeaufforderung hier öffnen" # Oder über cd-Befehl: cd "C:\Pfad\zu\Ihren\Bildern"

3. Dokumentation erstellen

# Basis-Erstellung wip # Erweiterte Optionen wip -t "Gernkogel Begehung" -d "06-08-2025" -l "Gernkogel, Österreich" -o "gernkogel_bericht.md"

4. Workflow für PDF-Export

Modularer Workflow für professionelle Berichte:
  • Markdown generieren: wip erstellt eine .md Datei mit Ihren Bildern
  • HTML wird glzt. generiert: mit wip wird glzt. mit .md Datei eine .html Datei erstellt. Diese Datei dient zur Überprüfung der Formatierung die vom pdf-style.css vorgegeben ist.
  • Manuell bearbeiten: Öffnen Sie die .md Datei in Ihrem Editor und passen Sie den Inhalt an! (Achtung: der Inhalt der .html Datei wird erst beim PDF-Export aktualisert!)
  • PDF exportieren: wip -p liest die aktuelle .md, generiert HTML neu und konvertiert zu PDF
Vorteile des modularen Workflows:
  • ✅ Markdown für einfache Bearbeitung
  • ✅ PDF für finale Präsentation und Druck
  • ✅ Professionelle Formatierung durch CSS-Styles
  • ✅ HTML wird automatisch aus aktuellem Markdown generiert
  • ✅ Alle manuellen Änderungen werden im PDF berücksichtigt

📸 Dateinamen-Schema

Unterstützte Formate

[beliebiger_name]___[longitude]_[latitude]___.[extension]
Schema-Erklärung:
🔵 Beliebig: Dateiname und Dateierweiterung
🔴 Koordinaten (optional): ___longitude_latitude___

Koordinaten-Format (falls vorhanden):
• Longitude: -180.0 bis +180.0 (Ost/West)
• Latitude: -90.0 bis +90.0 (Nord/Süd)
• Trennzeichen: Genau drei Unterstriche ___ vor und nach den Koordinaten
• Koordinaten-Trenner: Ein Unterstrich _ zwischen Longitude und Latitude

✅ Korrekte Beispiele

001_start___13.202000_47.316875___.jpg (mit Koordinaten)
aussicht___13.205002_47.315930___.png (mit Koordinaten)
geotagged_bruecke___13.207121_47.316326___.jpg (mit Koordinaten)
overview.jpg (ohne Koordinaten - wird trotzdem verarbeitet)

❌ Falsche Beispiele

foto___999.999_999.999___.jpg (ungültige Koordinaten)
start_13.202_47.316.jpg (falsches Trennzeichen-Schema)
bild___13.202_47.316.jpg (fehlende Unterstriche vor/nach Koordinaten)

Automatische Bildunterschriften

Das System generiert automatisch Bildunterschriften:
geotagged_vlcsnap-2025-08-06-13h15m45s450___13.205002_47.315930___.jpg
→ "Aufnahme - Koordinaten: 13.205002°E, 47.315930°N"

Verarbeitung aller Bilder

Alle Bilder werden verarbeitet: Das System verarbeitet jetzt alle Bilder im Ordner, unabhängig davon, ob sie Zeitstempel oder Koordinaten haben.

✅ Bilder MIT Zeitstempel

Bilder mit Datum/Zeit im Dateinamen
Vollständige Verarbeitung:
• Sortierung: Chronologisch nach Aufnahmezeit
• Caption: Mit Zeitstempel und Elevation
• Koordinaten: Falls verfügbar
• Distanz: Berechnung möglich

✅ Bilder OHNE Zeitstempel

Bilder ohne Datum/Zeit im Dateinamen
Basis-Verarbeitung:
• Sortierung: Werden zuerst angezeigt
• Caption: Aus Dateinamen generiert
• Koordinaten: Falls verfügbar
• Distanz: Nicht in Berechnung einbezogen
Wichtige Hinweise zur Bildverarbeitung:
  • Alle Bilder: Werden in die Dokumentation aufgenommen
  • Sortierung: Bilder ohne Zeitstempel erscheinen zuerst
  • Bilder mit Zeitstempel: Werden chronologisch sortiert
  • Koordinaten: Bleiben optional (für Distanzberechnung)
  • Caption-Generierung: Funktioniert für alle Bildtypen

🔧 Befehlsoptionen

📋 Alle Parameter mit Beispielen

📝 -t

wip -t "Alpenwanderung 2025" wip -t "Meine Bergtour"
Titel des Dokuments
Standard: "Wegdokumentation"

📅 -d

wip -d "15-03-2025" wip -d "06-08-2025"
Datum im Format DD-MM-YYYY
Standard: aktuelles Datum

📍 -l

wip -l "Berchtesgadener Alpen" wip -l "Gernkogel, Österreich"
Standort/Gebiet der Wanderung
Standard: "Wandergebiet"

📄 -o

wip -o "mein_bericht.md" wip -o "gernkogel_dokumentation.md"
Ausgabe-Markdown-Datei
Standard: "walk_documentation.md"

🎨 -T

wip -T scientific wip -T simple wip -T "C:\mein_template.md"
Template-Format auswählen
Werte: default, scientific, simple, Dateipfad

🗜️ -c

wip -c wip -c -q 90
Bildkomprimierung aktivieren
Reduziert Bildgröße, erstellt .backup Dateien

📏 -m

wip -m 1.5 wip -m 3.0 -c
Maximale Bildgröße in MB
Standard: 2.0, Bereich: 0.5-10.0

⚡ -q

wip -q 95 wip -q 75 -c
JPEG-Qualität nach Komprimierung
Standard: 85, Bereich: 1-100

🔍 --dry-run

wip --dry-run wip --dry-run -t "Test"
Testlauf ohne Dateierstellung
Zeigt Bildordnung und Verarbeitung

📖 --help, -h

wip --help wip -h
Kurze Kommandozeilen-Hilfe anzeigen
Kompakte Parameter-Übersicht

🌐 --help-browser

wip --help-browser
Ausführliche Hilfe im Browser öffnen
Öffnet diese Hilfeseite

📄 -p

wip -p wip -p -i my_file.md
Markdown zu PDF konvertieren
Konvertiert bestehende Markdown-Datei zu PDF über wkhtmltopdf
Beispiele:
wip -p → liest walk_documentation.md
wip -o "my_report.md" -p → liest my_report.md
wip -p -i "custom.md" → liest custom.md

📥 -i

wip -i "mein_bericht.md"
Eingabe-Markdown-Datei für PDF
Standard: verwendet -o Wert oder "walk_documentation.md"
Hinweis: Nur Markdown-Dateien (.md) können verarbeitet werden. Das System validiert automatisch die Dateiendung.

💡 Parameter-Kombinationen

Wichtige Hinweise:
  • Reihenfolge: Parameter können in beliebiger Reihenfolge verwendet werden
  • Kurzform: Alle Parameter verwenden das -x Format
  • Anführungszeichen: Verwenden Sie " " bei Leerzeichen in Werten
  • Pfade: Verwenden Sie absolute oder relative Pfade für -T
  • PDF-Modus: -p überschreibt alle anderen Parameter

🚀 Häufige Kommando-Kombinationen

Standard-Dokumentation erstellen:

wip -t "Meine Wanderung" -l "Alpenregion"

Wissenschaftliche Dokumentation mit Komprimierung:

wip -T scientific -t "Forschungswanderung" -l "Nationalpark" -c -q 95

Minimale Dokumentation für schnelle Berichte:

wip -T simple -t "Kurzer Rundgang" -o "quick_report.md"

PDF-Export nach Bearbeitung:

# 1. Markdown erstellen wip -t "Meine Wanderung" -l "Alpen" # 2. Markdown bearbeiten (manuell) # 3. Zu PDF konvertieren wip -p -i "walk_documentation.md"

🎨 Template-System

📋 Verfügbare Templates

Standard-Template: Das System kommt mit einem vorkonfigurierten default.md Template im templates/ Ordner.

📄 default.md

Standard-Template (wird automatisch verwendet)
Vollständige Wegdokumentation
Enthält alle wichtigen Elemente:
  • Zusammenfassung und Statistiken
  • Methodikbeschreibung
  • Foto-Dokumentation
  • Schlussfolgerungen
  • Koordinaten-Anhänge

🔧 Eigenes Template verwenden

Benutzerdefinierte Templates: Sie können eigene Template-Dateien erstellen und mit dem -T Parameter verwenden.
wip -T "C:\Pfad\zu\meinem_template.md"
Beispiel:
wip -T "C:\Templates\mein_wanderbericht.md"

💡 Template-Variablen

Verfügbare Platzhalter in Templates:
  • {title} - Dokument-Titel
  • {date} - Datum (DD-MM-YYYY)
  • {location} - Standort/Gebiet
  • {content} - Generierte Foto-Dokumentation
  • {total_images} - Anzahl Bilder
  • {total_distance} - Gesamtdistanz in km
  • {coordinates_list} - Koordinaten-Liste
  • {coordinate_bounds} - Min/Max Koordinaten

📁 Template-Ordner

Installationspfad: templates/default.md
Benutzerdefinierte Templates: Können überall gespeichert werden, verwenden Sie den vollständigen Pfad mit -T.

# 1. Markdown erstellen und bearbeiten wip -t "Gernkogel Begehung" -l "Gernkogel, Österreich" -c # 2.   • Schritt 1: wip generiert Markdown mit Bildern und Koordinaten

  • Schritt 3: wip -p konvertiert zu PDF mit professioneller Formatierung

🆘 Häufige Probleme und Lösungen

Problem: "Keine Bilder mit Koordinaten gefunden"

Lösung: Überprüfen Sie das Dateinamen-Format
✅ Korrekt: bild___13.202000_47.316875___.jpg
❌ Falsch: bild_13.202_47.316.jpg

Problem: "Kein Startbild gefunden"

Lösung: Ein Bild muss mit 001_ beginnen
Benennen Sie Ihr Startbild um: 001_start___koordinaten___.jpg

Problem: "Befehl nicht gefunden"

Lösung: System neu installieren oder PATH prüfen
# Installation prüfen dir "C:\Program Files\walk_image_processor" # PATH prüfen echo %PATH%

Problem: PDF-Export funktioniert nicht

Lösung: wkhtmltopdf installieren
# Download von: https://wkhtmltopdf.org/downloads.html # Nach Installation neu starten # Dann: wip -p funktioniert

Problem: PDF-Formatierung nicht korrekt

Lösung: CSS-Styles werden automatisch angewendet
# Das System kopiert automatisch pdf_styles.css # Bilder werden nach Orientierung skaliert (Portrait: 180mm, Landscape: 110mm) # Seitenumbrüche vor "Fotodokumentation" und "Anhänge"

Problem: Falsche Dateiendung bei -i Parameter

Lösung: Der -i Parameter akzeptiert nur Markdown-Dateien (.md)
# Korrekt: wip -p -i "mein_bericht.md" # Falsch (wird abgelehnt): wip -p -i "mein_bericht.html" wip -p -i "mein_bericht.pdf"
Automatische Validierung: Das System prüft die Dateiendung und zeigt eine Fehlermeldung bei falschen Formaten.

📸 .walk-image

Bildformatierung und Skalierung
Automatische Bildanpassung:
• Portrait: max-height: 230mm (1 pro Seite)
• Landscape: max-height: 110mm (2 pro Seite)
• Abstand: 10px zum Caption, 25pt oben
• Seitenumbrüche: Automatisch verhindert

📝 .image-container

Bild-Container-Zentrierung
Horizontale Zentrierung:
• Ausrichtung: Zentriert alle Bilder
• Breite: 80% der Seitenbreite
• Abstände: Keine zusätzlichen Margins

💬 em (Caption)

Bildunterschriften-Formatierung
Caption-Styling:
• Abstand: 0px oben (10px von Bild), 40px unten
• Breite: 80% (passt zu Bildern)
• Stil: Kursiv, zentriert, 12pt

📄 .page-break

Seitenumbrüche erzwingen
Manuelle Seitenumbrüche:
• Verwendung: Vor "Fotodokumentation" und "Anhänge"
• CSS: page-break-before: always
• Abstand: 20px Margin oben
Wichtige CSS-Eigenschaften:
  • Bildgrößen: Automatisch nach Orientierung optimiert
  • Abstände: 10px Bild-Caption, 40px Caption-nächstes Bild
  • Seitenumbrüche: Automatisch vor wichtigen Sektionen
  • Schriftgrößen: 12pt Basis, 14pt Überschriften, 16pt Haupttitel
  • Farben: Professionelle Blau-Grau-Palette (#3498db, #2c3e50)
